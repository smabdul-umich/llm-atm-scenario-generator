# ATM Training Scenario Generation and Evaluation

This repository contains scripts for generating and evaluating synthetic **Air Traffic Management (ATM)** training scenarios using LLMs and automated evaluation metrics.

## Overview

The workflow consists of three main steps:

1. Define prompts and few-shot examples
2. Generate synthetic training scenarios using Azure OpenAI
3. Evaluate generated scenarios using GEval metrics
4. Visualize evaluation results

---

# Files

## `prompts.py`

Contains the **system prompt** and **few-shot examples** used for generating ATM training scenarios.

These prompts guide the LLM to produce realistic and structured scenarios suitable for ATM training exercises.

---

## `gen_scenarios.py`

Generates training scenarios from prompts stored in a CSV file using **Azure OpenAI**.

The script reads prompts from the input CSV, generates scenarios with the LLM, and writes the results back to a CSV file.

### Expected Input CSV Columns

| Column                          | Description                                                           |
| ------------------------------- | --------------------------------------------------------------------- |
| Prompt                          | The prompt used to generate a training scenario                       |
| Scenario                        | The generated or manually written scenario                            |
| 1 if LLM generated, 0 otherwise | Indicates whether the scenario was generated by the LLM               |
| Few-shot to skip                | Indicates whether the row should be skipped during few-shot selection |

### Run

```bash
python gen_scenarios.py --input input_scenarios.csv --output output_scenarios.csv
```

If `--output` is not provided, the script may overwrite the input file.

---

## `eval.py`

Evaluates ATM training scenarios using **GEval metrics** with a custom rubric and evaluation steps.

The script reads scenarios from an input CSV file, runs evaluations for all scenarios in a single test run, and writes the results to a CSV file.

### Expected Input CSV Columns

| Column   | Description                             |
| -------- | --------------------------------------- |
| Prompt   | The prompt associated with the scenario |
| Scenario | The training scenario to evaluate       |

### Run

```bash
python eval.py --input input_scenarios.csv --output eval_results.csv
```

Evaluation results include scores across multiple rubric dimensions.

---

## `eval_viz.ipynb`

Jupyter notebook used for **visualizing GEval evaluation results**.

The notebook loads the evaluation spreadsheet exported from the DeepEval platform after running `eval.py`.

### Purpose

* Visualize scores across five rubric dimensions
* Compare performance of generated vs. human-written scenarios
* Explore evaluation distributions and trends

### Input

The notebook expects the **evaluation spreadsheet exported from the DeepEval platform**.

---

# Typical Workflow

1. Define prompts and few-shot examples in `prompts.py`
2. Generate scenarios:

```bash
python gen_scenarios.py --input input_scenarios.csv --output generated_scenarios.csv
```

3. Run evaluation:

```bash
python eval.py --input generated_scenarios.csv --output eval_results.csv
```

4. Visualize results using:

```
eval_viz.ipynb
```
